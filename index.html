📁 Terraform Project Structure
A simple guide for your terminal

Why this guide? Many people (even experienced engineers) get confused about how to organize Terraform files. This guide explains each file in plain English — step by step, without fancy jargon.

📦 What is a "project structure"?

It's just a way to name your files and arrange folders so you don't get lost. Good structure = you can find things, reuse them, and avoid mistakes.

📄 The 4 essential files (keep them together in one folder)

File nameWhat it does (plain English)Example
main.tfThe main recipe. It lists what to build (servers, containers, networks).resource "docker_container" "web" { ... }
variables.tfDefines placeholders (like "I will need a name" without saying the name).variable "container_name" { type = string }
outputs.tfShows results after building (e.g., "Your IP is 192.168.1.5").output "ip" { value = docker_container.web.ip }
terraform.tfvarsReal values for the placeholders (actual names, passwords, ports).container_name = "my-nginx"
🧁 Easy analogy: A cake recipe.
• main.tf = ingredients + oven
• variables.tf = "you can choose any filling"
• terraform.tfvars = you write "apple filling"
• outputs.tf = "cake ready at 3:00 PM"

🗂️ Optional folders (for bigger projects)

When you have more than 5–6 files, you can organize like this:

my-terraform-project/
│
├── main.tf
├── variables.tf
├── outputs.tf
├── terraform.tfvars
│
├── modules/               # Reusable components
│   └── webserver/
│       ├── main.tf
│       ├── variables.tf
│       └── outputs.tf
│
└── environments/          # Different setups: dev / prod
    ├── dev/
    │   └── terraform.tfvars
    └── prod/
        └── terraform.tfvars
  • 📦 modules/ : When you copy/paste the same code twice → turn it into a module.
  • 🌍 environments/ : When you want to test changes before applying to the real/live system.

💻 How to create this on any OS (Linux / macOS / Windows)

Open your terminal and type:

# 1. Create a new folder for your project
mkdir ~/my-first-terraform
cd ~/my-first-terraform

# 2. Create the 4 main files
touch main.tf variables.tf outputs.tf terraform.tfvars

# 3. (Optional) create a modules folder
mkdir modules

🚀 Working example (Docker + Nginx)

Copy these examples to see it actually work (requires Docker installed).

main.tf

terraform {
  required_providers {
    docker = {
      source = "kreuzwerker/docker"
    }
  }
}

resource "docker_image" "nginx" {
  name = "nginx:latest"
}

resource "docker_container" "web" {
  name  = var.container_name
  image = docker_image.nginx.name
  ports {
    internal = 80
    external = var.external_port
  }
}

variables.tf

variable "container_name" {
  description = "Name of the Docker container"
  type        = string
  default     = "my-nginx"
}

variable "external_port" {
  description = "Port on your laptop to access Nginx"
  type        = number
  default     = 8080
}

outputs.tf

output "container_id" {
  value = docker_container.web.id
}

output "access_url" {
  value = "http://localhost:${var.external_port}"
}

terraform.tfvars

container_name = "my-dev-nginx"
external_port   = 8081

🧪 Commands to run (in order)

📌 From your project folder ~/my-first-terraform :

terraform init     # Downloads the Docker plugin (run once)
terraform plan     # Shows what will happen (no changes yet)
terraform apply    # Actually creates the Docker container (type "yes")
terraform destroy  # Removes everything when you're done

✅ Summary – remember this

📌 One folder. Four files. That's all you need to start.
• main.tf = What to build
• variables.tf = Placeholders
• terraform.tfvars = Real values
• outputs.tf = Results

🔧 Next steps (any OS)

# Install Terraform (Linux / macOS / Windows via package managers)
# For Linux Mint / Ubuntu:
sudo apt update && sudo apt install terraform -y

# Install Docker (Linux example)
sudo apt install docker.io -y
sudo systemctl start docker
sudo usermod -aG docker $USER   # (log out and back in)

# Create the example files and run "terraform init" + "terraform apply"
